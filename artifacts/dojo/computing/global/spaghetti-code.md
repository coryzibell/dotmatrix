# Spaghetti Code

**Topic:** computing
**Scope:** global
**Date:** 2025-11-30

## Knowledge

**Spaghetti code** is code that works but is poorly structured, tangled, and hard to maintain. Like trying to trace one noodle through the bowl.

Characteristics:
- Excessive goto statements or deeply nested conditionals
- No clear structure or separation of concerns
- Difficult to follow control flow
- Changes in one place break things elsewhere

Related food metaphors:
- **Lasagna code**: too many layers of abstraction
- **Ravioli code**: too many small, disconnected pieces
- **Pizza code**: flat but covered in random toppings (features)

## Why It Matters

- Technical debt accumulates fast
- Makes onboarding new devs painful
- Refactoring becomes risky
- Often the result of "just ship it" culture without cleanup

## Quiz Question

What's the term for code that works but is poorly written and hard to maintain?

## Answer

Spaghetti code.
