## Breaking Changes

### Dependency Updates

The primary change is updating transitive dependencies:
- `dirs-sys` updated to 0.5.0
- `windows-sys` updated to 0.59.0

**Impact:** Minimal. These are internal dependency updates with no API changes in dirs itself.

## API Stability

The dirs API remains stable. No function signatures changed between 5.0.1 and 6.0.0.

## Context: What Changed in 5.0

For reference, 5.0 introduced:
- **New:** `config_local_dir()` for non-roaming configuration on Windows
- **Breaking:** `executable_dir()` no longer depends on `$XDG_DATA_HOME`

If you're on 5.0.1, these are already handled.
